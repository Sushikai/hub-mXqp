阅读06-stock-bi-agent代码回答问题
1. 什么是前后端分离？
1.1前后端分离是一种软件开发架构模式，将前端（用户界面）与后端（业务逻辑、数据接口）独立开发、部署和运行。
前端负责界面展示和用户交互，通过 HTTP 请求调用后端提供的 API 获取数据。
后端只提供 API 接口（如 RESTful API），处理业务逻辑、数据库操作等，并返回 JSON 等格式的数据。
项目中：
后端：main_server.py 启动 FastAPI 服务，提供 /docs 下的接口（如用户、聊天、股票等路由），以及底层的股票 API。
前端：./demo/ 目录下的 Streamlit 应用，通过调用后端 API 实现聊天、股票查询、用户管理等功能。

2. 历史对话如何存储，以及如何将历史对话作为大模型的下一次输入？

2.1存储方式
历史对话记录保存在 ./assert/conversations.db（SQLite 数据库）中。
后端提供了对话切换、历史记录管理功能，说明每条对话会话（session）的消息会被持久化存储。
将历史对话作为大模型下一次输入的方式
使用 AdvancedSQLiteSession（依赖 openai-agent 服务）

2.2输出
从数据库中读取当前会话的历史消息，限制取最近的 N 条（例如最后 10 条），将这些消息拼接或按对话格式（如 user/assistant 交替）组装成新的输入，连同当前用户问题一起发送给大模型。
实现步骤：
用户发送新消息 → 后端从数据库获取该会话最近 10 条消息。
将这些历史消息与当前用户输入一起构造为完整的对话上下文（例如 OpenAI Chat Completion 格式的 messages 数组）。
调用大模型接口，模型基于完整上下文生成回复。
将本次用户输入和模型回复追加存储到数据库中。
