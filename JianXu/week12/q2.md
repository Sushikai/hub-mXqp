## 一、什么是前后端分离？
前后端分离是一种软件架构模式：**前端（界面）和后端（业务逻辑/数据）分别独立部署，通过 HTTP API 通信**，互不耦合。


## 二、历史对话如何存储 & 如何作为大模型的下一次输入？
1.业务库 sever.db
用户消息：append_message2db(session_id, "user", content)
助手完整回复：append_message2db(session_id, "assistant", full_reply)
作用：用于前端展示、会话记录查询、业务持久化。
2.Agent 会话库 conversations.db
通过 AdvancedSQLiteSession(session_id, ...) 管理。
Runner.run_streamed(..., session=session) 会自动：
读取该 session_id 下的历史消息；
在本轮结束后写入本轮 user/assistant 消息。
作用：用于给大模型恢复上下文。
