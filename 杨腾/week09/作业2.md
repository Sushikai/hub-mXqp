## 系统总体架构

### 系统整体流程如下：

``` text
用户问题  
→ 意图识别（LLM分类）  
→ 路由分发  
→ 知识库检索（Elasticsearch）  
→ 上下文构建  
→ LLM生成最终答案 
``` 

------------------------------------------------------------------------

## 三、意图识别模块设计

``` text
1. 分类目标
类别	含义
1	普通问题
2	机器学习
3	大模型
2. Prompt设计
请判断用户问题属于哪一类：

1 普通问题（非机器学习/大模型）
2 机器学习
3 大模型
``` 
------------------------------------------------------------------------

## 四、系统流程实现
``` text
Step1：用户输入
query = 用户问题
Step2：意图识别
intent = llm_classify(query)
Step3：路由分发
if intent == 1:
    return llm(query)

elif intent == 2:
    docs = search("ml_kb", query)
    return rag(query, docs)

elif intent == 3:
    docs = search("llm_kb", query)
    return rag(query, docs)
``` 

------------------------------------------------------------------------

## 五、RAG核心流程
``` text
- 1. 检索（Retrieval）

基于 Elasticsearch：

倒排索引（Inverted Index）
BM25相关性评分算法

示例：
es.search(index="ml_kb", query=query)

2. 上下文构建
context = "\\n".join([doc["content"] for doc in docs])

3. 生成（Generation）
请基于以下知识回答问题：

【知识】
{{context}}

【问题】
{{query}}
```
