## 1. BERT 文本分类和实体识别有什么关系？分别使用什么 loss？

### 1.1 二者关系
这份代码使用的是**多任务联合学习**：
- 同一条用户输入文本，同时做两件事：
  - 意图分类（句子级）
  - 槽位填充/实体识别（token级）
- 两个任务共享同一个 BERT 编码器，再接两个任务头。


### 1.2 分别使用什么 loss
本项目中两个任务都使用 CrossEntropyLoss：
- 意图分类：seq_loss = CrossEntropyLoss(seq_output, seq_label_ids)
- 实体识别：	oken_loss = CrossEntropyLoss(active_logits, active_labels)

## 2. 多任务训练 loss = seq_loss + token_loss 的坏处，以及不平衡如何处理

### 2.1 直接相加的坏处
loss = seq_loss + token_loss（main.py 第 56 行）虽然简单，但有以下风险：
1. 默认等权重不一定合理：两个任务难度和 loss 量级可能不同。
2. 某个任务可能主导训练：loss 更大的任务对梯度影响更强，导致另一任务学不好。
3. 任务之间可能有梯度冲突：一个任务的优化方向可能伤害另一个任务。
4. NER 常见类别不平衡：O 类太多时，模型容易偏向预测 O。

### 2.2 出现训练不平衡时的常见处理
1. **加权损失**：loss = alpha * seq_loss + beta * token_loss，在验证集上调 lpha/beta。
2. **动态权重**：例如 uncertainty weighting、GradNorm 等。
3. **任务采样/训练调度**：交替训练、调采样比例、为不同任务设不同学习率。
4. **任务内再平衡**（尤其 NER）：加权交叉熵、Focal Loss、重采样等。
